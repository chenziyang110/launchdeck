## Planning Inputs

- Implementation target: `F:/github/launchdeck`.
- Locked boundary: one authoritative local lifecycle control plane shared by CLI, MCP, Skill, and generated Plugins.
- Compatibility boundary: preserve CLI `schemaVersion: 1`; version Agent protocol, config, registry, journal, package, and components independently.
- Safety boundary: configured-project, low-risk, local stdio Agent operations only; unknown/external ownership is inspect-only; uncertain mutations stop and reconcile without replay.
- Evidence boundary: real stdio, real host, exact OS/version/architecture/scope/build receipts; a missing cell withholds only its claim.
- UI contract: not applicable; no UI-bearing tasks are planned.
- Project cognition advisory: the cognition database was unavailable during planning, so implementation must recheck live touched owners and stop on contradiction.

- Setup, Foundational, US1, US2, US3, US4, and Polish are execution phases, not delivery deferrals.
